CONKW - The default grabbers



This page lists all the grabbers available
OSHI Grabber This is the universal system grabber. Not necessarily extremely optimized, it can get data from most systems. From a top-like UI to the brand of your RAM modules. Beware on particularly old and slow systems, this grabber (if all extractors are enabled) can drain your system. Proc Grabber This grabber gather its data from the /proc and /sys filesystems on Linux. It is fast and reasonably optimized. Sys Grabber The Sys grabber is a universal system grabber, just using standard Java stuff. It provides basic CPU, load, memory and filesystems metrics. Open Weather Map Grabber This grabber extracts the weather forecast from one call API of the openweathermap website. You need to register for a free account in order to use it. File Grabber This grabber is not configured by default. It allows conkw to read data from any file that respects the properties format with some restrictions added. Look up the documentation for more information.
