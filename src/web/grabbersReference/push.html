This is a sample PushToEmiGrabber. It pushed the "sys" grabber to the "test_emi" grabber. The metrics you see below are comiong from the "test_emi" grabber, pushed there by the PushToEmiGrabber. Obviously there is no point in doing that to your instance, but it illustrates how you can easily push metrics from one conkw instance to another. You can have for example all of your computers that send metrics to a central instance, monitoring your entire set of machines in one place.

Have a look at the default configuration file to see this example. Or look at the documentation.
